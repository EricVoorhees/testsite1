Welcome to Vision AI Docs

Vision AI is a desktop-first, voice-powered intelligent assistant designed to help you interact with your system in real time. Built with privacy and performance at its core, Vision runs locally, responds instantly, and automates your workflow like never before. This documentation covers setup, features, architecture, and modules.

This is the official documentation for Vision AI. It covers everything from setup to advanced configuration, feature usage, and updates.

Architecture

Vision AI is built on a modular architecture designed for performance, modularity, and local control:

  • Vision Core – Handles all logic flow, task routing, context awareness, and memory.
  • Voice Engine – Uses STT (Speech to Text) and TTS (Text to Speech) powered by Whisper and ElevenLabs.
  • UI Panel – A custom PyQt5 frontend that persistently displays responses, logs, and voice output.
  • Memory & Config – JSON-based memory and settings system that tracks usage, preferences, and user-defined commands.
  • OCR & Camera Modules – Optional tools for reading text on screen or analyzing live camera input.

Installation & Setup

To install Vision AI locally:

  1. Ensure Python 3.11+ is installed
  2. Clone the GitHub repo: git clone https://github.com/vision-ai/assistant
  3. Install dependencies: pip install -r requirements.txt
  4. Run the main file: python main.py

First launch will prompt access to microphone and camera if enabled. All configuration is handled in settings.json.

Features

  • Wake word activation – Always listening for your custom phrase (e.g., “Vision”)
  • Contextual voice commands – Ask questions, run code, open apps, or automate your desktop
  • Script triggers – Say “run my dev build” or “launch recording setup” and Vision executes it
  • Real-time OCR – Use your voice to ask “What does this say?” while hovering over text
  • Secure & local – Nothing is sent to the cloud unless explicitly configured
  • Voice activation with context-awareness
  • Live memory and task history
  • System automation and scripting
  • Optional OCR & visual recognition

Modules

Each module is built to be independent but works with Vision Core:

  • Vision Core: Manages command parsing, memory loading, and output routing
  • Command Engine: Maps voice to shell/script or action. Can run bash, PowerShell, or Python logic
  • Speech Engine: Handles all TTS/STT (Whisper for input, ElevenLabs for voice out)
  • UI System: PyQt5 based dock that overlays persistently and shows live log feedback
  • OCR Pipeline: Uses Tesseract or other tools to extract visible text from apps

Each module is individually documented with configuration examples and integration logic. More detail coming soon.

FAQ

  • Does Vision work offline? – Yes. All logic is local. Only TTS (if using ElevenLabs) or GPT need internet.
  • Can I write my own commands? – Yes. Add scripts or shell commands to the command map in commands.py.
  • Can I change the voice? – Yes. Use any ElevenLabs voice or fallback to offline options.
  • Does Vision record me? – No. It listens only for your wake word and you control when to activate.
